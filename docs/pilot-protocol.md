| 版本 | v0.2 |
|------|------|
| 適用階段 | Phase 1A 攝點建置（首次）＋ 例行季配 / 月配拍攝 |
| 場域 | 大雪山土肉桂試驗 4 樣區（115-12-1 成熟林 + 117-28-1 / 8-1-1 / 8-1-10 幼齡林）|
| 對應 ADR | [ADR-0005 固定攝點 SOP](decisions/0005-fixed-camera-station-sop.md)（攝點規格與建置/重訪細節之權威來源）|

> v0.2 重點：從「隨意 4 方位試拍」改為「**固定攝點**重複造訪」（試驗對照前提，見 ADR-0005）。新增攝點建置 SOP、campaign（按樣區分批）、成熟林特例。

> **抽樣（依 [ADR-0006](decisions/0006-image-sampling-design.md)）**：Pilot 每處理組**固定追蹤 5–10 株**（驗流程＋估變異數），正式階段 10–15 株；**系統抽樣涵蓋大小範圍**（如每處理抽編號 1、5、10…），不挑漂亮的、不每期換株。葉片摘取依三階段（建模/例行/重驗），非每株每季。拍攝技術細節見 [field-imaging-sop.md](field-imaging-sop.md)。

## 壹、拍攝前準備

### 一、人員（依 ADR-0005 分工）

| 階段 | 角色 | 負擔 |
|------|------|------|
| 攝點建置（首次）| 公會技術員 + 合作社人員 | 設備經費由計畫支應、工時公會＋合作社分擔 |
| 例行拍攝 | 合作社人員（受訓後兼任）+ 記錄 | 人在現場，邊養護邊拍最省 |

> 例行拍攝最少 1 人可兼拍攝＋記錄（語音備忘輔助）；建置階段建議 ≥ 2 人。

### 二、設備清單

#### (一) 拍攝設備

| 裝備 | 備註 |
|------|------|
| 相機 | 手機（iPhone 13+／Android 旗艦）或 DSLR；幼齡林標準 2.5 m，成熟林需廣角 |
| 三腳架（穩定型）× 2 | 架於攝點標樁正上方，確保高度/角度可重現 |
| 尺度物（葉片必備）| A4 方格紙、5 元硬幣、10 元硬幣、25 cm 鋼尺（任一入鏡）|
| 色卡 | X-Rite ColorChecker 或 24 色卡（建議）|
| 紀錄 | 平板／手機表單 + 紙本備援；**攜帶 `fixed_camera_station.csv`** |
| 遮光 | 太陽傘（葉片近拍製造柔光，避免直射過曝）|

#### (二) 攝點建置設備（首次造訪，估 ≈ NT$33,400 / 480 株）

| 裝備 | 數量 | 備註 |
|------|------|------|
| 金屬細木樁（30 cm）| 每株 3 標樁（北/東/葉片區）| 打入地面、露出 3 cm |
| 油漆筆（紅/黃/藍）| 各組 | 北=紅、東=黃、葉片區=藍（高反差易識別）|
| 雷射測距儀（30 m）| 1 | 校正攝距 |
| 羅盤／指南針 | 2 | 記錄 azimuth_deg |
| 捲尺（10 m）| 2 | 量攝距與相機高 |

### 三、出發前檢核

- [ ] 樣樹已對應 `tree_registry.csv`（tree_id 既有碼，如 `115121-C0-001`）
- [ ] **重複造訪**：`fixed_camera_station.csv` 已帶在身邊；確認各株標樁完好（紅/黃/藍油漆樁）
- [ ] 設備電量 > 80%、儲存空間充足（單樣區一日約數百張，預留 5 GB）
- [ ] 紙本紀錄表印好（[`metadata/field-record-form.md`](../metadata/field-record-form.md)）
- [ ] 對拍攝員與記錄員口頭過一次 SOP（5 分鐘）

## 貳、現場拍攝步驟

### 一、建立採集場次（Campaign）

依 ADR-0005：**按樣區分批、同一樣區當日完成**（控光照一致）；四樣區地理分散，不要求同日全拍。

1. 抵達樣區，記錄員建立 campaign：`campaign_id = {site_id}_{季別}`（如 `8-1-1_114Q3`）、`capture_date`、天氣、開始時間。
2. 季別對齊官方季配：採收期月配、非採收期季配。
3. 校正白平衡（色卡或灰卡拍一張）。
4. **攝點 baseline 對齊**：到每株第一個攝點先拍 1 張，與 `fixed_camera_station` 既有 baseline 比對視角是否一致。

### 二、攝點建置 SOP（首次造訪，新增）

> 完整步驟見 [ADR-0005 §四-(一)](decisions/0005-fixed-camera-station-sop.md)。摘要：

1. 確認 / 掛樹牌（tree_id）。
2. 打 3 核心標樁：北側（紅）、東側（黃）、葉片區（藍）；標樁與樹幹水平距離＝攝距（幼齡林 2.5 m），羅盤記 azimuth_deg。
3. 量基準：初始株高、DBH（或地徑）、雷射測距校正攝距。
4. 拍 baseline 影像作為「攝點對齊參考」。
5. 登記 `fixed_camera_station.csv`：tree_id、station_id、azimuth_deg、distance_to_trunk_m、camera_height_m、created_at、created_by。

### 三、攝點重訪拍攝（每株核心 ≥ 3 攝點）

取代 v0.1「隨意 4 方位」。每株依 `fixed_camera_station` 拍攝：

| 攝點 station_label | 內容 | 攝距 | 高度／角度 |
|--------------------|------|------|-----------|
| `north_side`（核心）| 北側全株：株高、冠幅、整株葉量、葉色 | 幼齡林 2.5 m | 1.5 m，水平 0° |
| `east_side`（核心）| 東側全株：交互驗證、W2 冠幅 | 2.5 m | 1.5 m，水平 |
| `leaf_macro`（核心）| 葉片近拍：葉面積/葉色/病斑/蟲咬 | 30–50 cm | 垂直紙面 90° |
| `top_view`（加值）| 樹冠俯拍：冠幅投影 | — | 伸縮桿/小梯架，俯拍 -90° |
| `pruning_site_close_up`（加值）| 修剪部位特寫：再生狀態 | ~50 cm | 含修剪切口入鏡 |

重訪操作：三腳架置標樁正上方（誤差 < 5 cm）→ 依 station 規格設高度/角度 → 固定 ISO（手機 ISO 200~400）→ 連拍 3 張取最佳 → 記錄實際攝距/相機高/station_id。

> **避免**：背光剪影、葉片反光、被他樹遮擋（偏移時記 `obstruction_flag`）。

### 四、成熟林特例（115-12-1，樹高 8–12 m）

依 ADR-0005 決議（樹高超出 2.5 m 框幅）：

1. **5–7 m 廣角拍樹冠為主**（north_side/east_side 改 5–7 m 攝距，記實際攝距）。
2. **主幹另近拍分兩張**（樹冠 + 主幹分張，不強求整株一張）。
3. 無人機俯拍列 v1 加值。
4. 矮化保留主幹 3–4 m 完成後，可回標準 2.5 m 攝距。

### 五、葉片近拍（每株 5~10 片代表性葉）

代表性標準（ADR-0005 #5）：樹冠中段成熟葉 5 片 + 含病徵 2~3 片。

1. 剪小枝（含完整葉）或原位拍；鋪 A4 方格紙當背景兼尺度物。
2. 葉片完整入鏡、邊緣留 1 cm；鏡頭垂直紙面；距 30~50 cm。
3. 確認尺度物（A4 邊框）≥ 70% 在畫面內。
4. 記錄：是否含色卡、健康／異常／病徵類型。

### 六、即時品質檢查

每樣區每拍 5 株，記錄員快檢最近影像：

- [ ] 整株是否完整入鏡（成熟林：樹冠 + 主幹分張是否齊）？
- [ ] 葉片有無模糊／過曝／過暗？尺度物完整？
- [ ] station_id / campaign_id 是否正確標記？

任一不過 → 當場補拍。

## 參、回辦公室處理

### 一、影像下載

```powershell
$src = "D:\DCIM\..."   # 相機記憶卡
$dst = "H:\我的雲端硬碟\2026 tree_multiscale_data\raw\canopy"
robocopy $src $dst *.jpg /XO
```

### 二、檔名重命名

依 [SPEC.md 第肆章](SPEC.md) 命名規則：

```
{station_id}_{YYYYMMDD}_{HHMM}_{seq}.{ext}
例：115121-C0-001.north_side_20260601_1430_001.jpg
```

### 三、metadata 整合

把現場表單彙整到 [`metadata/image_metadata.template.csv`](../metadata/image_metadata.template.csv) 欄位格式，存 `metadata/image_metadata.csv`。樣樹/量測資料可由 [`scripts/etl_longformat_to_entities.py`](../scripts/etl_longformat_to_entities.py) 由官方調查長表產生對齊版本。

### 四、品質彙整

每 campaign 寫一份簡短檢討（記於 Obsidian 工作筆記 `H:\我的雲端硬碟\secondbrain\10-專案\TreeVision\工作筆記.md` 或本 repo `logs/`）：拍攝張數（規劃 vs 實際）、不合格率、規格/流程漏洞、瓶頸、修訂建議。

## 肆、Phase 1A 評估

首批攝點建置（建議先 115-12-1 ~123 株為 1A，幼齡林三區 ~358 株為 1B）後檢核：

- [ ] 單株重訪拍攝 ≤ 5 分鐘？建置 ≤ 30 分鐘？
- [ ] 影像品質合格率 ≥ 90%？
- [ ] 成熟林 5–7 m 廣角是否足以解析葉/病徵？
- [ ] `fixed_camera_station.csv` / metadata 欄位是否需增刪？

→ 結論回寫 ADR-0005 或新 ADR；通過後進入例行季配 / 月配拍攝。

## 伍、安全注意事項

| 風險 | 預防 |
|------|------|
| 山區跌傷 | 兩人同行、防滑鞋 |
| 蜂／蛇／虎頭蜂 | 蜂螫應急包、不穿鮮豔衣 |
| 中暑 | 9~11 點與 14~16 點為主拍攝時段，避開正午 |
| 雷雨 | 看雷達回波即時撤退；金屬三腳架避免高地 |
| 山區無訊號 | 雙人離線地圖 |
