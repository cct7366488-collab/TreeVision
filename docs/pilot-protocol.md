# 試拍協定 SOP（Pilot Field-Shoot Protocol）

| 版本 | v0.1 |
|------|------|
| 適用階段 | Phase 1：第一次試拍（建議在正式蒐集前先做 1 場域 × 3~5 棵樹的試拍） |
| 目的 | 驗證資料規格的可行性、調整尺度物與光照配置、發現規格漏洞 |

---

## 壹、試拍前準備

### 一、人員

| 角色 | 人數 | 職責 |
|------|------|------|
| 拍攝員 | 1~2 | 操作相機、選擇拍攝位置 |
| 記錄員 | 1 | 填寫 metadata 表單、計時、檢查影像品質 |
| 樹木編號 / 嚮導 | 1 | 對應樣樹、處理現場狀況 |

> 試拍階段建議至少 2 人。後續正式拍攝可由 1 人兼任拍攝＋記錄，但需用語音備忘錄輔助。

### 二、設備清單

| 類別 | 裝備 | 備註 |
|------|------|------|
| 相機 | 手機（iPhone 13+ 或 Android 旗艦）或 DSLR / 無反 | 主要設備；試拍可同時帶兩種比較成果 |
| 鏡頭 | 廣角（樹冠用，等效 24~35 mm） + 微距（葉片用，等效 50~100 mm） | 手機可用內建廣角 + 1× 鏡頭 |
| 尺度物（葉片必備） | A4 方格紙（≥ 1 張）、新台幣 5 元硬幣（≥ 5 個）、新台幣 10 元硬幣（≥ 5 個）、25 cm 鋼尺 | 任選一種放在每張葉片影像中 |
| 色卡 | X-Rite ColorChecker 或 24 色色卡 | 建議；無則需確保白平衡準確 |
| 樹木標示 | 樹牌、噴漆筆 / 油漆膠帶 | 預先編號 `T0001`~ |
| 紀錄 | 平板 / 手機（用 Google Form） + 紙本表單備援 | 兩者並用 |
| 其他 | 雨布、太陽傘（避免直射陽光導致過曝） | 視天氣 |

### 三、出發前檢核

- [ ] 樣本場域已確認（GPS 邊界、入山許可若需要）
- [ ] 樣樹已預編號並對應到 `tree_registry.csv`
- [ ] 設備電量 > 80%、儲存空間充足（樹冠 ≥ 10 MB／張、葉片 ≥ 15 MB／張，請預留 5 GB）
- [ ] 紙本紀錄表印好（[`metadata/field-record-form.md`](../metadata/field-record-form.md)）
- [ ] 數位 metadata 表單測試過（Google Form 或試營運的網頁表單）
- [ ] 對拍攝員與記錄員口頭過一次 SOP（5 分鐘）

## 貳、現場拍攝步驟

### 一、建立場次（Session）

1. 抵達場域，記錄員填表：`site_id`、`session_id = {site_id}_{YYYYMMDD}_{HHMM}`、`device_id`、天氣、能見度、溫度、開始時間。
2. 校正白平衡（用色卡或灰卡拍一張）。
3. 拍攝場域全景（不歸檔，僅留存於 session 紀錄）。

### 二、單棵樹拍攝（每棵 ~10 分鐘）

#### A. 樹冠影像（4 張）

每棵樹拍攝 4 個方位：北、南、東、西。

| 步驟 | 內容 |
|------|------|
| 1 | 距離樹幹約 5~10 m（依樹高調整，整棵樹冠完整入鏡） |
| 2 | 鏡頭水平、後仰約 10~30°（讓樹冠主體佔畫面 60% 以上） |
| 3 | 設定 ISO ≤ 400，光圈 f/8（手機自動模式即可） |
| 4 | 對焦在樹冠中心，連拍 2~3 張取最佳 |
| 5 | 記錄員勾選表單：`view`=`north`/`south`/`east`/`west`、確認檔名規則 |

> **避免**：背光（陽光在樹後方造成剪影）、葉片反光、被其他樹冠完全遮擋。

#### B. 葉片近拍（5~10 片）

每棵樹採集 5~10 片**代表性樣本**（健康 + 異常各半）。

| 步驟 | 內容 |
|------|------|
| 1 | 用枝剪剪下小枝（含 1~3 片完整葉），或在原位拍 |
| 2 | 鋪 A4 方格紙當背景（兼尺度物） |
| 3 | 葉片**完整入鏡**，邊緣與紙邊預留 1 cm 安全距 |
| 4 | 鏡頭垂直紙面（避免梯形變形） |
| 5 | 距離 30~50 cm，用微距鏡或手機 1× 鏡 |
| 6 | 確認尺度物（A4 邊框）至少 70% 在畫面內 |
| 7 | 記錄員填表：`leaf_seq`、是否有色卡、是否健康／異常／病徵類型 |

> **避免**：葉片折角、強反光（避免直射陽光、用太陽傘製造柔光）、尺度物部分出畫。

### 三、即時品質檢查

每 5 棵樹結束後，記錄員打開最近的 5 張影像快速檢查：

- [ ] 樹冠是否完整入鏡？
- [ ] 葉片有無模糊、過曝、過暗？
- [ ] 尺度物是否完整可見？
- [ ] EXIF 拍攝時間是否合理？

任一項不通過 → 當場補拍。

## 參、回辦公室處理

### 一、影像下載

```powershell
# 路徑：H:\我的雲端硬碟\2026 tree_multiscale_data\raw\
# 樹冠
$src = "D:\DCIM\..."   # 相機記憶卡
$dst = "H:\我的雲端硬碟\2026 tree_multiscale_data\raw\canopy"
robocopy $src $dst *.jpg /XO   # /XO = 只複製比目標新的
```

### 二、檔名重命名

依 [SPEC.md 第肆章](SPEC.md) 命名規則：

```
{site_id}_{tree_id}_{capture_type}_{YYYYMMDD}_{HHMM}_{view}_{seq}.{ext}
```

可用 `scripts/rename_raw_images.py`（待實作；試拍階段先手動或用 ExifTool 批次）。

### 三、metadata 整合

把現場紙本＋數位表單的內容彙整到 [`metadata/image_metadata.template.csv`](../metadata/image_metadata.template.csv) 的格式，存成 `metadata/image_metadata.csv`。

### 四、品質彙整

每場次寫一份簡短檢討（紀錄於 [Obsidian 工作筆記](../../G:/我的雲端硬碟/secondbrain/10-專案/TreeVision/工作筆記.md) 或本 repo `logs/`）：

| 項目 | 檢核 |
|------|------|
| 拍攝張數（規劃 vs 實際） |  |
| 品質不合格率 |  |
| 規格漏洞發現（例：尺度物不夠大、view 定義模糊） |  |
| 流程瓶頸（例：單棵樹耗時超過預估） |  |
| 規格／流程修訂建議 |  |

## 肆、試拍後評估

完成 1 場 × 3~5 棵樹後（預估半天 + 整理 1 天），檢核：

- [ ] 一棵樹的拍攝時間 ≤ 10 分鐘？
- [ ] 影像品質合格率 ≥ 90%？
- [ ] metadata 表單欄位是否需要增刪？
- [ ] 尺度物選擇是否需要更換？
- [ ] 規格 v0.1 有無漏洞？

→ 整理結論寫入新 ADR（`docs/decisions/0002-pilot-findings.md`）。
→ 通過評估後，正式進入 Phase 1 大規模拍攝。

## 伍、安全注意事項

| 風險 | 預防 |
|------|------|
| 山區跌傷 | 兩人同行、穿防滑鞋 |
| 蜂、蛇、虎頭蜂 | 隨身帶蜂螫應急包、不穿鮮豔衣 |
| 中暑 | 9~11 點與 14~16 點為主拍攝時段，避開正午 |
| 雷雨 | 看雷達回波即時撤退；金屬三腳架避免高地 |
| 山區無訊號 | 雙人離線地圖（Maps.me / 山林日誌） |
