### 階段 1：資料收集 + 標註（2026-05 ~ 2026-08）

| 任務 | 目標數量 | 備註 |
|------|----------|------|
| 試拍協定確認 | 1 場域、3 棵樹 | 確認尺度物、光照、view 設計 |
| 樹冠影像 | ≥ 500 張 | 涵蓋 4 視角 × 多時段 |
| 葉片近拍 | ≥ 1000 片 | 含尺度物，分散健康／異常樣本 |
| 標註團隊培訓 | 2 名標註者 | 1 主標 + 1 複標 |
| 標註完成 | 80% 影像 | 剩餘留作 hold-out |
| 跨標註 IoU 報告 | ≥ 0.55（病徵）、≥ 0.85（樹冠） | 達標才釋出訓練 |

### 階段 2：模型 v0 訓練（2026-07 ~ 2026-09）

| 任務 | 達成指標 |
|------|----------|
| CanopySeg baseline | mIoU ≥ 0.65 |
| LeafInst baseline | mAP@50 ≥ 0.70 |
| LeafDefect baseline | 各類 IoU ≥ 0.45 |
| 推論服務 (FastAPI 包裝) | 單張 ≤ 60 秒（CPU） |
| 評估報告 | 各模型 model card |

### 階段 3：平台 MVP（2026-08 ~ 2026-11）

#### Sprint 規劃

| Sprint | 期程 | 主要產出 |
|--------|------|----------|
| S1 | 2026-08 | 後端 skeleton + DB schema + auth |
| S2 | 2026-09 | 影像上傳 + metadata 表單 + 品質檢查 |
| S3 | 2026-09 | 推論佇列 + worker 整合 + 結果儲存 |
| S4 | 2026-10 | 樹木資料庫頁 + 影像列表 + 單影像詳情 |
| S5 | 2026-10 | 儀表板（場域 + 樹木層級） |
| S6 | 2026-11 | PDF 報表（單影像／單樹／場域） |
| S7 | 2026-11 | 內部 UAT 修正、效能調校 |

## 伍、風險里程碑檢核

每階段結束需檢核：

- [ ] 達成標註 IoU 門檻？
- [ ] 模型驗收指標達成？
- [ ] 端到端流程可走通？
- [ ] 安全 / 隱私審查通過？
- [ ] 文件齊全（README + 規格 + 標註指南 + API doc）？

任一項未達門檻 → 進入緩衝期，不前進至下一階段。
