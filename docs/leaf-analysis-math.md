> **核心命題**：模型本身不「測量」公分，它只在像素世界裡圈出形狀。把像素換回真實世界的公分（幾何基準）、把 RGB 換回可比的葉色（輻射基準），這兩件事完全由「前置處理」與「拍攝當下的物理條件」決定。**因此拍攝品質是整條判釋鏈的座標基準與輻射基準，直接決定數值的可信度上限。**

## 壹、整體判釋鏈總覽

每一個輸出數值都沿下列鏈逐級計算；鏈上任一級的輸入品質不足，後面所有數值一起失真：

```
原始影像
  │
  ├─[前置處理]── 幾何校正(mm/pixel)、色彩校正(白平衡/色卡)、品質閘(模糊/過曝)
  │
  ├─[分割/偵測模型]
  │     ├ LeafInst        → 每片葉 polygon、bbox、尺度物、mm_per_pixel
  │     ├ LeafDefect      → 每片葉 病斑/黃化/壞死/破洞 多標籤遮罩
  │     ├ CanopySeg       → 樹冠 綠/黃/褐 像素分類
  │     └ PlantStructure  → 全株/樹冠/主幹 遮罩、株頂株底像素
  │
  ├─[單影像幾何量化]── polygon面積→cm²、PCA長寬軸、像素比值→比例指標
  │
  ├─[彙總]── 每株 × 每場次（多張平均）
  │
  └─[推估與統計]── 採收量估計、葉片大小分佈、新老葉比、處理組 ANOVA
```

對照 SPEC 之模型規格見 [SPEC §伍-一~三](SPEC.md)；對照 14 項響應指標見 [ADR-0004 §二](decisions/0004-experimental-response-metrics.md)。

## 貳、影像前置處理的數學原理

前置處理解三個獨立問題：**尺寸對不對（幾何）、顏色準不準（輻射）、影像清不清（品質）**。

### 一、幾何校正——像素換算公分（mm/pixel）

#### (一) 基本比例尺

所有面積、長度、株高數值的根。用畫面中已知真實尺寸的尺度物反推每像素代表幾公釐：

```
s = ref_size_mm / ref_size_px        （s 單位：mm/pixel）
```

例：A4 方格紙長邊 297 mm，影像中量得 1,270 px，則 `s = 297 / 1270 ≈ 0.234 mm/pixel`（即 [SPEC §伍-二](SPEC.md) 輸出之 `mm_per_pixel`）。葉面積換算：

```
leaf_area_cm² = area_px × (s / 10)²
```

`s` 為**平方項**——此平方在第伍章成為最關鍵的誤差放大來源。

#### (二) 透視變形與單應性校正

相機光軸若不垂直於葉片平面，畫面產生梯形變形（perspective foreshortening）：同一片葉，近端與遠端的 mm/pixel 不同，全幅失去單一比例尺。傾斜角 θ 在傾斜軸方向引入縮短因子 cos θ，造成的面積偏差約：

```
面積相對誤差 ≈ (1 / cos θ) − 1
```

| 傾斜角 θ | 面積偏差 |
|:---:|:---:|
| 5° | ≈ 0.4% |
| 15° | ≈ 3.5% |
| 25° | ≈ 10.3% |
| 35° | ≈ 22% |

校正法：以 A4 方格紙四個已知直角頂點解**單應矩陣（homography）H**，將斜拍紙面拉正回正射視角，使全幅回到單一 mm/pixel：

```
[x']     [h11 h12 h13][x]
[y']  ≈  [h21 h22 h23][y]
[w']     [h31 h32  1 ][1]      →  (x'/w', y'/w') 為校正後座標
```

> **拍攝對應**：SOP「鏡頭垂直俯視、方格紙四角完整入鏡」即為此校正的控制點需求。缺角或無格線→ H 無解，比例尺只能單點近似，斜拍偏差無法事後移除。

### 二、色彩校正——RGB 轉為可跨時序比較的訊號

葉色指標本質是 RGB 通道的比值或差值，對白平衡與曝光極度敏感。

#### (一) 兩個葉色指標的數學形式（[ADR-0004](decisions/0004-experimental-response-metrics.md) B1、B2）

```
GLI（Green Leaf Index）       = (2G − R − B) / (2G + R + B)        範圍 −1~1
TGI（Triangular Greenness）  = G − 0.39R − 0.61B                  葉綠素代理
```

#### (二) 為何白平衡必須鎖定、不可自動

自動白平衡（AWB）逐張改變三通道增益 `(g_R, g_G, g_B)`，觀測值 `R' = g_R · R_true`。g 每張跳動→ GLI、TGI 分子分母同步漂移→ 同一片葉不同場次算出不同葉色，**處理組差異被拍攝雜訊淹沒**。鎖定「日光」WB ＋每場次拍**色卡/灰卡**，提供已知參考可事後反解 g、把影像拉回同一輻射基準。

#### (三) 為何「寧可欠曝、絕不過曝」是不可逆的

感測器過曝時通道值被截斷（clip）於 255：

```
若 G → 255（飽和），TGI = 255 − 0.39R − 0.61B 失去真實綠度資訊
病斑/黃化的色彩對比一併溢白消失，無法復原
```

欠曝僅把訊號線性壓低（可乘回增益救回）；過曝是**截斷（非線性、資訊湮滅）**。前者可逆、後者不可逆——此即 SOP 將過曝列為鐵律之根本。

### 三、品質閘——模糊與過曝的客觀篩除

入庫管線（`scripts/ingest_images.py`）以兩個量化指標自動擋掉不可用影像：

#### (一) 模糊偵測（清晰度）

以 Laplacian 響應的變異數衡量邊緣銳利度：

```
clarity = Var( ∇²I )            ∇² 為 Laplacian 算子
clarity < 門檻 → 判定模糊、退件
```

葉脈、病斑、葉緣皆高頻邊緣訊號；模糊使 `Var(∇²I)` 塌陷，模型無從辨識。

#### (二) 過曝偵測

計算葉片遮罩內任一通道 ≥ 250 的像素占比：

```
overexp_ratio = #{ pixel : max(R,G,B) ≥ 250 } / #{ leaf pixels }
overexp_ratio > τ（如 0.02）→ 退件、待補拍
```

### 四、模型輸入正規化

通過品質閘後 resize 至模型輸入尺寸（CanopySeg 長邊 1536、LeafInst 長邊 2048）並做通道標準化 `(x − μ)/σ`。**mm/pixel 必須在原始解析度上計算後再隨縮放因子換算**，比例尺才不被 resize 破壞——故原圖須保最高解析度（[SPEC §肆-二](SPEC.md)：葉片近拍 ≥ 4000×3000），縮放只能往下、不能無中生有。

## 參、葉片尺度的分析模型與數學

葉片尺度兩模型串接：**LeafInst（圈出每片葉）→ LeafDefect（在每片葉上圈病徵）**。

### 一、LeafInst：葉片實例分割 + 尺度物偵測

#### (一) 任務與輸出

實例分割（instance segmentation）對每片葉輸出獨立 polygon。輸出含每片葉的 `polygon`、`bbox`、`area_px`，加尺度物類型與 `mm_per_pixel`（[SPEC §伍-二](SPEC.md)）。

#### (二) 從 polygon 算幾何量

1. **面積**——多邊形頂點以鞋帶公式（shoelace）：

```
area_px = ½ |Σ (xᵢ·yᵢ₊₁ − xᵢ₊₁·yᵢ)|
leaf_area_cm² = area_px × (s/10)²
```

2. **長/寬/長寬比**——對 polygon 點集做主成分分析（PCA），協方差矩陣兩特徵向量即葉片主軸與次軸（等價於旋轉最小外接矩形之長短邊）：

```
leaf_length_mm = (主軸投影最大跨距) × s
leaf_width_mm  = (次軸投影最大跨距) × s
leaf_aspect    = leaf_length / leaf_width
```

### 二、LeafDefect：病徵的多標籤分割

#### (一) 任務性質

多標籤語義分割——**同一像素可同時屬於多個病徵**（如壞死區邊緣同時是病斑），故為 4 個獨立二元遮罩（lesion / chlorosis / necrosis / hole），非互斥分類。輸入為 LeafInst 切出的單葉 ROI。

#### (二) 病徵比例指標（皆以該葉像素為分母）

```
lesion_ratio       = lesion 像素     / 葉片像素
chlorosis_ratio    = chlorosis 像素  / 葉片像素
necrosis_ratio     = necrosis 像素   / 葉片像素
hole_ratio         = hole 像素       / 葉片像素
total_defect_ratio = (任一病徵聯集像素) / 葉片像素
```

> 分母為「葉片像素數」而非「影像像素數」→ 病徵率為無因次比值、**不需尺度物即可算**；但分子分母都依賴葉緣與病斑邊緣的清晰度，故仍要求高解析、銳利、不過曝。

#### (三) 衍生健康分數（[ADR-0004](decisions/0004-experimental-response-metrics.md) 衍生指標）

```
health_score = 100 − 100 × (w1·yellowing_ratio + w2·disease_lesion_ratio + w3·pest_damage_ratio)
clamp 至 [0,100]
權重（cinnamon_v0_1）：w1=0.30, w2=0.40, w3=0.30
```

權重抽於 `models/LeafDefect/config.yaml`，不寫死於程式碼，可隨樹種校正（[ADR-0001 議題 3](decisions/0001-open-questions.md)）。

### 三、葉色與葉綠素代理（在體影像，CanopySeg 路徑）

CanopySeg 於全株/樹冠影像先把像素分綠/黃/褐，再於遮罩內統計 RGB 求 GLI、TGI（§貳-二），並得：

```
green_ratio     = veg_green 像素 / (veg_green + veg_yellow + veg_brown)
yellow_ratio    = veg_yellow 像素 / 同上
brown_ratio     = veg_brown 像素 / 同上
yellowing_ratio = 黃化葉占比（對應 ADR-0004 B3）
```

## 肆、採收量、葉片結構、分佈的推估

把幾何量往上彙整為經營有意義的數值。

### 一、葉片數量（leaf_count_visible，A1）

LeafInst 在單張全株影像偵測到的葉片實例數。

關鍵限制：此為「**可見葉數**」非「真實葉數」，被遮擋者必低估：

```
N_visible = N_true × v(視角, 遮擋)，  v < 1 且未知
```

`v` 無法逐株精確求得，但只要**固定攝點**使同株各場次 `v` 近似常數，則生長量抵消之：

```
ΔN = N_visible(t2) − N_visible(t1) ≈ v · (N_true(t2) − N_true(t1))
```

此即 [ADR-0006 §五](decisions/0006-image-sampling-design.md) 採「以生長量為反應變數 + 重複量測」的數學依據——**不要求拍到絕對真值，只要求每次偏差一致**。對拍攝者而言：絕對值不準可接受，**攝點漂移才致命**。

### 二、葉片結構（單片幾何 + 全株彙整）

#### (一) 單片結構

平均葉面積、葉長、葉寬、長寬比、葉色（來源見第參章）。`mean_leaf_area_cm2 = mean(leaf_area_cm²)`（對應 [ADR-0004](decisions/0004-experimental-response-metrics.md) A4）。

#### (二) 總葉面積估計（A5，LAI 代理）

```
total_leaf_area_estimate_cm² = leaf_count_visible (A1) × mean_leaf_area_cm² (A4)
```

葉面積指數（LAI）之單株代理量，反映光合面積與生長勢。

### 三、採收量（鮮重/乾重）估計——非破壞性推估

官方靠採收後天秤量枝葉鮮重/乾重（破壞性）。TreeVision **不直接替代**，以影像量推估後迴歸對接真值（[SPEC §拾參-三](SPEC.md)）。

#### (一) 推估原理

葉生物量與總葉面積近似線性（單位面積葉重大致穩定）：

```
估計乾重 Ŵ = β₀ + β₁ × total_leaf_area_estimate
            （等價：Ŵ = A_total / SLA，SLA = 比葉面積 cm²/g）
```

β₁（或 SLA）由**建模期摘葉實測**（[ADR-0006](decisions/0006-image-sampling-design.md) 葉片三階段之「建模階段」）對影像估計值迴歸校正得到。

#### (二) 可信度驗收（必過此線方可使用）

[ADR-0004 §評估指標](decisions/0004-experimental-response-metrics.md) 與 [SPEC §玖](SPEC.md) 之硬門檻：

```
影像估計 vs 官方人工量測（DBH、樹高、枝葉乾重）：Pearson r ≥ 0.7
葉計數 MAPE ≤ 15%、葉面積 MAPE ≤ 8%
```

通過前葉片指標暫用摘葉量測；通過後切換「拍照讓模型讀、不再摘葉」（[ADR-0006](decisions/0006-image-sampling-design.md) 過渡期設計）。

### 四、葉片大小分佈（distribution）

不只看平均，更看分佈——對偵測處理是否令葉片趨大/趨齊最敏感。LeafInst 輸出每片葉面積後：

```
分佈統計：mean、std、p25、p50（中位數）、p75
變異係數 CV = std / mean        （葉片整齊度）
```

施肥有效常表現為**中位數上移 + CV 下降**；此差異在平均值上可能被極端值掩蓋、在分佈/百分位上才顯現。故 SOP 要求「取代表葉、嚴禁挑漂亮的」——**選擇偏差會直接扭曲分佈估計**。

### 五、新葉/老葉比例（A2、A3，再生強度）

```
new_leaf_count（A2）= LeafInst × 葉色分類（嫩葉紅褐/淡綠）
new_leaf_ratio（A3）= A2 / A1
```

依賴葉色判讀，故同樣受白平衡/曝光影響——回到色彩校正。

### 六、處理組統計檢定（最終目的）

逐株逐場次彙總 → 每處理組彙總 → 餵入重複量測混合模型（[ADR-0006 §五](decisions/0006-image-sampling-design.md)）：

```
反應變數 ~ 修剪 × 施肥 × 季別 + (1|樣木) + (1|樣區)
```

輸出 F 值、p 值、effect size、交互作用熱力圖。**檢力警示**（[ADR-0006 §檢力分析](decisions/0006-image-sampling-design.md)）：每處理 n=5 連大效應檢力僅 20%，故固定追蹤 10–15 株、以生長量為反應變數提升檢力。

## 伍、誤差傳遞——拍攝品質決定數值上限

三個最關鍵的誤差放大機制：

### 一、比例尺誤差在面積上「加倍」

因 `A = N × s²`，誤差傳遞：

```
σ_A / A = √[ (σ_N/N)² + (2·σ_s/s)² ]
```

`s` 前係數為 **2**——尺度物量測 5% 誤差，葉面積即約 10% 誤差，再傳進採收量估計。**故尺度物必須清晰、完整入鏡、與葉共面、垂直俯視**。此為平方放大，非龜毛。

### 二、曝光截斷使色彩指標「歸零失效」

過曝為資訊湮滅（§貳-二-(三)），葉色、黃化、病斑一併喪失且不可逆。色彩類指標（B1~B5、新葉比）可信度**上限**由此鎖死。

### 三、攝點漂移使「生長訊號被遮擋偏差淹沒」

§肆-一已述：固定攝點令未知遮擋因子 `v` 為常數而可在生長量中抵消；攝點漂移則 `v` 改變，`ΔN` 混入拍攝幾何變化而非樹的真實生長，訊噪比崩潰。

## 陸、拍攝動作 ↔ 保護的數值（對應表）

| 拍攝動作（務必） | 數學上保護的數值 | 違反的後果 |
|---|---|---|
| 尺度物清晰、完整、與葉共面、垂直俯視 | `mm_per_pixel` → 葉面積、株高、採收量 | 比例尺誤差**平方放大**（5%→10% 面積誤差） |
| 鏡頭垂直紙面、方格紙四角入鏡 | 單應校正（去梯形變形） | 傾斜 25° → 面積偏差 ~10%，無法事後校正 |
| 白平衡鎖「日光」、每場次拍色卡 | GLI/TGI、黃化率、新葉比 | 葉色跨場次不可比，處理組差異被淹沒 |
| 寧可欠曝、絕不過曝（必要 EV −0.3） | 所有葉色與病徵指標 | 過曝＝資訊**不可逆**湮滅 |
| 三腳架 + 鎖焦 + 連拍取最佳 | 清晰度閘 `Var(∇²I)` | 模糊使葉脈/病斑無法辨識、整批退件 |
| 固定攝點（同樁/距/高/角） | 葉計數生長量 ΔN、所有時序比較 | 遮擋偏差 `v` 漂移，生長訊號失真 |
| 取代表葉、嚴禁挑漂亮的（5–10 片） | 葉面積分佈、CV、平均 | 選擇偏差扭曲分佈與處理效應估計 |
| 葉片平鋪、不重疊、不折角 | LeafInst 逐片實例分割 | 重疊→實例黏連，逐片量測失敗 |
| 全株完整入鏡、不切頂/切基 | 株高 C1、冠幅 C2、株型 | 株高量測直接失準 |
| 幼齡林用單色背板、成熟林選單純方位 | CanopySeg/PlantStructure 分割品質 | 背景雜物→分割誤判→葉量/葉色連帶錯 |
| 解析度開最高（葉片 ≥ 4000×3000） | 病斑/葉脈像素數 | 像素不足→細病徵辨識不出，resize 救不回 |

## 柒、結論要點

一、**拍攝決定數值可信度上限，非模型決定**。模型只在像素世界圈形狀；把像素換成公分（比例尺/垂直俯視）、把 RGB 換成可比葉色（白平衡/色卡/不過曝）兩個基準握在拍攝者手上，且多數錯誤（過曝、斜拍、攝點漂移）事後不可逆。

二、**最該死守三件事**（按數學放大程度）：(一) 尺度物（面積平方放大）；(二) 不過曝（資訊湮滅）；(三) 固定攝點（生長訊號命根）。

三、**葉片摘採量測先多後少**：建模期需摘代表葉建 ground truth 並迴歸校正（驗到 r ≥ 0.7）；驗收通過後切換「拍全株、模型讀數、不再摘葉」，僅每季每處理留 2–3 株重驗防漂移。
