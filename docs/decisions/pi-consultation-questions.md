## 壹、回答方式

請針對每題：

1. 在「回應」欄填寫決議或建議
2. 若需更多時間思考，標記「待補」並指明預估回覆時間
3. 對於無法決議的議題，可改為「TreeVision 自行假設並標明」

回填後，TreeVision 將依回應更新對應 ADR 為 `accepted` 並落地至 SPEC v0.2。

## 貳、A 級問題（**阻塞 Phase 1 啟動**，最高優先）

### 一、樣區編號標準

| 項目 | 內容 |
|------|------|
| **問題** | 報告書內文「115-12-1」與位置圖「115-012-02」實際採何者作為 site_id？ |
| 影響 | 所有 schema、檔名、metadata 的 site_id 命名規範 |
| 建議 | 採用**地圖編號 115-012-02**（最新、含填零格式 8 碼，便於程式排序） |
| **回應** | ✅（2026-06-13 由試驗資料定案）實際資料採人類標籤 `115-12-1 / 117-28-1 / 8-1-1 / 8-1-10` ＋短碼 `115121/117218/811/8110`；地圖「115-012-02」**未被任何資料檔採用**。site_id 採 dash 標籤為準。**附帶更正**：報告書第二章「1-2-3」應為「8-1-1」之誤（第三章與實際資料均為 8-1-1）。 |

### 二、樣樹是否已實體編號

| 項目 | 內容 |
|------|------|
| **問題** | 4 樣區共約 300 株樣樹是否已掛樹牌或標記？編號格式為何？ |
| 影響 | tree_id 對應規則、Phase 1 攝點建置工序 |
| 建議 | 若已編號 → 沿用官方代碼；若無 → 採 `<site_id>.T<NNNN>` 格式（如 `115-12-1.T0001`） |
| **回應** | ✅（2026-06-13 由試驗資料定案）**已編號**。實際碼＝`<樣區>-<處理>-<序號>`（例 `115121-C0-001`），多幹個體加莖序 `-s2/-s3`；每處理格 1–30 號。TreeVision tree_id **改對齊此既有碼**，不另採 `.T0001`。 |

### 四、20 年生樣區的對照組

| 項目 | 內容 |
|------|------|

### 六、修枝矮化「適度強度」量化標準

| 項目 | 內容 |
|------|------|
| **問題** | 報告書描述「剪除頂芽 + 適度疏除側枝」，PPTX 教材給「人工採收建議離地 1~1.5 m 截幹」。試驗實際採用：(a) 截幹高度 = ?、(b) 保留分枝數 = ?、(c) 保留葉量比例 = ?、(d) 是否需保留頂芽？ |
