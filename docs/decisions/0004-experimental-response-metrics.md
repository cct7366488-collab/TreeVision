## Context

### 一、原 v0.1 健康指標的不足

[SPEC v0.1 第陸章](../SPEC.md) 設計的健康指標：

- `health_score_global` / `health_score_species`（0~100 標量）
- `health_grade`（A~E 字母分級）

問題：

1. **單一標量無法支援試驗對照**。試驗需要多維響應變數（葉量、葉色、生長、健康），用一個分數融合會掩蓋處理組的差異。
2. **未量化葉片數量**，這是土肉桂試驗的核心響應變數（採收量直接相關）。
3. **未量化株型生長**（株高、冠幅），這是修剪試驗的主響應變數。
4. **未量化新葉 / 老葉比例**，無法評估再生強度。

### 二、官方試驗的響應變數

[references/cinnamon-trial-context.md §肆-一](../../references/cinnamon-trial-context.md) 列出官方記錄項目：

| 項目 | 量測方式 | TreeVision 角色 |
|------|---------|---------------|
| 樹高、DBH（每季） | 人工 | 影像可校正 |
| 枝葉鮮重 / 乾重 | 採收後天秤 | 不替代（破壞性） |
| 精油含量 / 成分 | 蒸餾 + GC-MS | 不替代（化學） |
| 土壤養分 / pH / 逕流 | 實驗室 | 不替代 |

> 官方未列、但試驗價值高的指標（**這就是 TreeVision 的補強空間**）：
> - 葉片計數
> - 新葉 / 老葉比例
> - 葉色（葉綠素代理指標）
> - 黃化葉占比
> - 病斑覆蓋率
> - 蟲咬痕跡覆蓋率
> - 葉面積指數（LAI 代理）
> - 樹冠投影面積
> - 株高（影像自動量測）
> - 修剪後新生分枝數

## Options

### 選項 A：保留 v0.1 健康分數，外掛試驗響應指標（雙軌）

| 屬性 | 內容 |
|------|------|
| 變更幅度 | 小 |
| 概念清晰度 | 中（「健康度」與「試驗響應」同時存在） |
| 與官方對接 | 中 |

### 選項 B：完全捨棄健康分數，純試驗響應變數

| 屬性 | 內容 |
|------|------|
| 變更幅度 | 中 |
| 概念清晰度 | 高 |
| 通用性 | 低（未來其他場域 / 樹種無「試驗」上下文時失去意義） |

### 選項 C：分層設計 — 核心響應指標 14 項 + 衍生健康指標 1 項

```
 ┌─────────────────────────────────────────────────────────┐
 │ 核心響應指標（14 項，基本量化資料）                          │
 │ ├─ 葉片產量類（5）                                         │
 │ ├─ 葉色 / 健康類（5）                                       │
 │ └─ 株型生長類（4）                                          │
 ├─────────────────────────────────────────────────────────┤
 │ 衍生健康指標（1 項，便於主管報表）                            │
 │ └─ health_score（由 14 指標加權合成）                       │
 └─────────────────────────────────────────────────────────┘
```

| 屬性 | 內容 |
|------|------|
| 變更幅度 | 中 |
| 概念清晰度 | 高（雙層分工：14 客觀指標 vs 1 主觀分數） |
| 通用性 | 高（14 指標獨立，未來其他場域可裁切） |
| 與試驗對接 | 高（直接餵入 ANOVA） |

---

## Decision

### 一、選擇 **選項 C**（14 核心指標 + 1 衍生健康分數）

### 二、14 項核心響應指標規格

#### (一) A 類：葉片產量類（5 項，對應採集頻度 + 修剪試驗）

| 序號 | 指標名 | 型別 | 範圍 | 計算來源 | 對應試驗 |
|------|-------|------|------|---------|---------|
| A1 | `leaf_count_visible` | INT | ≥ 0 | LeafInst on whole-plant 影像；單張影像可見葉片實例數 | 1, 2, 3 |
| A2 | `new_leaf_count` | INT | ≥ 0 | LeafInst + 葉色分類（嫩葉紅褐 / 淡綠） | 1, 2 |
| A3 | `new_leaf_ratio` | DOUBLE | 0~1 | A2 / A1 | 1, 2 |
| A4 | `mean_leaf_area_cm2` | DOUBLE | ≥ 0 | LeafInst polygon area + 尺度物校正 | 2, 3 |
| A5 | `total_leaf_area_estimate_cm2` | DOUBLE | ≥ 0 | A1 × A4 | 1, 2, 3 |

> 試驗代號：1 = 採集頻度試驗（TreeVision 補強，官方未列）／ 2 = 修剪試驗 ／ 3 = 施肥試驗

#### (二) B 類：葉色 / 健康類（5 項，對應施肥試驗 + 整體健康監測）

| 序號 | 指標名 | 型別 | 範圍 | 計算來源 | 對應試驗 |
|------|-------|------|------|---------|---------|
| B1 | `green_index_mean` | DOUBLE | -1~1 | 全株遮罩內 RGB 統計：GLI = (2G - R - B) / (2G + R + B) | 3 |
| B2 | `chlorophyll_proxy_tgi` | DOUBLE | — | 葉片區 RGB 統計：TGI = G - 0.39R - 0.61B | 3 |
| B3 | `yellowing_ratio` | DOUBLE | 0~1 | LeafInst + 葉色分類「黃化」類別占比 | 3 |
| B4 | `disease_lesion_ratio` | DOUBLE | 0~1 | LeafDefect 模型輸出：病斑遮罩面積 / 總葉面積 | 3 |
| B5 | `pest_damage_ratio` | DOUBLE | 0~1 | LeafDefect 模型輸出：蟲咬痕跡面積 / 總葉面積 | 3 |

#### (三) C 類：株型生長類（4 項，對應修剪試驗）

| 序號 | 指標名 | 型別 | 範圍 | 計算來源 | 對應試驗 |
|------|-------|------|------|---------|---------|
| C1 | `plant_height_cm` | DOUBLE | ≥ 0 | 全株分割 + 尺度物校正 | 2 |
| C2 | `crown_projected_area_cm2` | DOUBLE | ≥ 0 | 俯拍 CanopySeg | 2 |
| C3 | `crown_height_ratio` | DOUBLE | 0~1 | （樹冠高 / 株高） | 2 |
| C4 | `branch_count` | INT | ≥ 0 | PlantStructure（v1+；v0 為 NULL） | 2 |

### 三、衍生健康指標（1 項）

| 指標 | 公式 | 範圍 |
|------|------|------|
| `health_score` | 1 - (B3 × w1 + B4 × w2 + B5 × w3) × 100 | 0~100 |

權重預設：

```yaml
health_weights:
  config_id: cinnamon_v0_1
  w1_yellowing: 0.30
  w2_disease: 0.40
  w3_pest: 0.30
```

> 配置檔位置：`models/LeafDefect/config.yaml`（沿用 ADR-0001 議題 3 既有路徑），新增 `cinnamon_v0_1` 條目。

### 四、計算範圍對齊

| 指標類別 | 量測單位 |
|---------|---------|
| A 類葉片產量 | **每株 × 每攝點 × 每場次**（單張影像） |
| B 類葉色健康 | **每株 × 每場次**（多張影像平均） |
| C 類株型生長 | **每株 × 每場次** |
| 衍生 health_score | **每株 × 每場次** |

> 場次 = `campaign`（見 ADR-0003）

### 五、儲存層對應

| 表 | 欄位範圍 |
|----|---------|
| `image_metric` | A1, A2, A4, B1, B2, B3, B4, B5, C1, C2, C3 |
| `treatment_response_summary` | A1~A5, B1~B5, C1~C4, health_score（每處理組 × 每場次彙總，供 ANOVA 使用） |

### 六、單位與精度

| 類別 | 精度 |
|------|------|
| 葉片計數 | 整數（`INT`） |
| 比例 | 4 位小數（`DOUBLE`） |
| 面積 cm² | 1 位小數 |
| 株高 cm | 1 位小數 |

## Consequences

### 一、SPEC 異動

- **第陸章 健康指標**全章重寫
- **第伍章 模型架構**：新增 PlantStructure 子節（C1~C4）
- 第柒章資料 schema：`image_metric` 新增 11 欄、新表 `treatment_response_summary`（見 ADR-0003）

### 二、模型訓練影響

| 模型 | 是否須重訓 |
|------|-----------|
| CanopySeg | 是（增加俯拍視角訓練） |
| LeafInst | 是（含葉齡分類分支：嫩葉 / 成熟葉） |
| LeafDefect | 是（病徵類別精細化：病斑 / 黃化 / 蟲咬） |
| PlantStructure（NEW） | 全新訓練 |

### 三、評估指標

| 模型 | 主指標 |
|------|-------|
| CanopySeg | mIoU ≥ 0.75 |
| LeafInst | mAP@50 ≥ 0.65、葉計數 MAPE ≤ 15% |
| LeafDefect | per-class IoU ≥ 0.50、影像級 ratio MAE ≤ 0.05 |
| PlantStructure | 株高量測 MAPE ≤ 10% |
| **試驗對照** | 14 指標 vs 官方人工量測的相關係數 r ≥ 0.7（DBH、樹高、枝葉乾重） |

### 四、報表異動

新增報表類型 `experiment_compare`：

```
[頁 1] 試驗概覽（4 樣區地圖 + 處理組分布）
[頁 2] 14 指標總覽（每指標一張趨勢圖：x = 場次日期，y = 指標值，多色線 = 處理組）
[頁 3] 修剪 × 施肥交互作用熱力圖（每指標一張）
[頁 4] ANOVA 結果表（F 值、p 值、effect size）
[頁 5] 多重比較結果（LSD / Tukey）
[頁 6] 影像 vs 官方人工量測的對照散點圖
```

## 未解決問題

| # | 問題 | 影響 |
|---|------|------|
| 1 | 「新葉 / 老葉」的葉色分類門檻是否需要實地校正？ | A2 / A3 計算 |
| 2 | 葉綠素代理指標是否需採用 hyperspectral 校正？v0 用 TGI 是否足夠？ | B2 設計 |
| 3 | 14 指標的權重（合成 health_score）是否需要由計畫主持人決議？ | health_weights config |
| 4 | C4 branch_count 是否要在 v0 試做？還是延後到 v1+？ | 模型架構規劃 |
| 5 | 指標精度是否需要按處理組調整（如施肥試驗對葉色精度要求更高）？ | 標註與訓練資源分配 |
