## Context

### 一、隨意拍攝 vs 試驗級拍攝的根本差別

[ADR-0004](0004-experimental-response-metrics.md) 的 14 響應指標**都依賴時序對照**（baseline vs 處理後第 N 場次）。
若拍攝姿勢不固定，即使同一棵樹兩次拍攝結果也會出現劇烈變動：

| 變動源 | 對指標的干擾 |
|--------|------------|
| 拍攝距離不同 | 影響葉計數（透視縮放） |
| 拍攝方位不同 | 改變可見葉片集合 |
| 拍攝高度不同 | 改變樹冠投影面積估計 |
| 拍攝時刻不同（光照） | 改變葉色 / 黃化指標 |
| 鏡頭焦距不同 | 改變尺度物校正基準 |

> **結論**：沒有固定攝點，試驗對照失效。**這是 SOP 的最重要前提**，不是可選項。

### 二、現行 [pilot-protocol v0.1](../pilot-protocol.md) 的不足

| 規範 | 現狀 | 缺口 |
|------|------|------|
| 樹冠 4 方位 | 寫了 north/south/east/west | **沒寫拍攝距離標準化、攝點地面標記、攝點 GPS 紀錄** |
| 葉片近拍 | 寫了 30~50 cm 距、A4 紙背景 | 寫得不錯，但「同一棵樹下次怎麼找到原本拍過的葉片」未說明 |
| 整株（whole plant） | **缺漏整章** | 試驗監測的核心 capture_type，必須補上 |

### 三、官方修剪技術 SOP 的觀察線索

[references/cinnamon-trial-context.md §陸](../../references/cinnamon-trial-context.md)（PPTX 教材）指出：
- 修剪後新生側枝會出現在「離切口 15~20 cm 範圍內」
- 健康評估的 5 個現場指標（葉片狀態、新芽分枝、傷口癒合、根部、整體樹勢）

> 這些**修剪後再生位置**是模型可學習的視覺線索。固定攝點若覆蓋修剪部位，模型可長期追蹤該樹的修剪反應。

## Options

### 選項 A：每樹單一攝點（最低成本）

| 屬性 | 內容 |
|------|------|
| 攝點數 | 1 / 樹 |
| 訊息覆蓋 | 不全（只有單一視角） |
| 拍攝時間 | 短 |
| 試驗效力 | 中（多時序仍可對照） |
| 適用 | 草根方案 |

### 選項 B：每樹 4 攝點（北 / 東 / 南 / 西，地面水平）

| 屬性 | 內容 |
|------|------|
| 攝點數 | 4 / 樹 |
| 訊息覆蓋 | 較全（葉量估計穩定） |
| 拍攝時間 | 中 |
| 試驗效力 | 中-高 |
| 適用 | 標準方案 |

### 選項 C：每樹 5 攝點（4 方位 + 1 俯拍）

| 屬性 | 內容 |
|------|------|
| 攝點數 | 5 / 樹 |
| 訊息覆蓋 | 全（含冠幅俯拍） |
| 拍攝時間 | 長（需伸縮桿 / 梯架） |
| 試驗效力 | 高 |
| 適用 | 高保真方案 |

### 選項 D：分層方案 — 核心攝點 + 加值攝點

| 屬性 | 內容 |
|------|------|
| 必拍攝點 | 2~3 / 樹（北側全株 + 葉片近拍 + 必要時尺度物驗證） |
| 加值攝點 | 機動方位、俯拍（隨設備可行性） |
| 試驗效力 | 高（核心指標可算，加值指標有就用） |
| 適用 | **務實平衡** |

---

## Decision

### 一、選擇 **選項 D**（分層方案 — 核心 + 加值）

### 二、核心攝點（必拍，每株 ≥ 3 個）

| Station ID | 名稱 | 用途 | 攝距 | 仰角 | 高度 | 主要支援指標 |
|-----------|------|------|------|------|------|-----------|
| `north_side` | 北側全株 | 株高、冠幅 W1、整株葉量、葉色 | **2.5 m** | 0° 水平 | 1.5 m（離地） | A1, A4, A5, B1, B2, B3, C1, C2 |
| `east_side` | 東側全株 | 與北側交互驗證；W2 量測 | 2.5 m | 0° | 1.5 m | A1, A4, A5, B1, C2 |
| `leaf_macro` | 葉片近拍 | 葉面積、葉色、病斑、蟲咬 | **30~50 cm** | 90°（垂直紙面） | 機動 | A4, B1~B5 |

> 為何選 2.5 m 攝距：幼齡林（1~3 m 樹高）可整株框入；15 年生成熟林需站遠至 5~7 m，須調整為**「整樹剛好框入」原則**，並紀錄實際攝距。
>
> 葉片近拍的攝點不固定樹身位置（每次採摘代表性葉片放 A4 紙拍），但 **拍攝姿勢固定**。

### 三、加值攝點（機動，視可行性）

| Station ID | 名稱 | 用途 | 拍攝方式 |
|-----------|------|------|---------|
| `top_view` | 樹冠俯拍 | 冠幅投影面積（C2 高保真） | 1.5~2 m 高伸縮桿 / 小梯架 / 無人機（v1+） |
| `south_side` | 南側全株 | 北側對照、抗光照偏差 | 同 north_side 規格 |
| `west_side` | 西側全株 | 同上 | 同 east_side 規格 |
| `pruning_site_close_up` | 修剪部位特寫 | 修剪後再生狀態 | 近距 50 cm，含修剪切口入鏡 |

### 四、固定攝點建立 SOP

#### (一) 攝點建置（一次性）

1. **挑選樣樹**（依 ADR-0003 編號規則 `<site_id>.T<NNNN>`）
2. **掛樹牌**（油漆膠帶或金屬牌，含 tree_id）
3. **建立攝點標樁**：
   - 用**金屬細木樁 + 油漆**（北、東、葉片區），打在地面 3 cm 露出
   - 油漆顏色：北 = 紅、東 = 黃、葉片區 = 藍（高反差易識別）
   - 樁與樹幹的水平距離 = 2.5 m（公尺尺實測）
   - 樁的方位用**手機指南針 / 羅盤**確認，記錄 azimuth_deg（方位角）
4. **量測樣樹基準**：
   - 初始株高（公尺尺垂直量測，記錄到 cm）
   - 初始 DBH（卡尺 / 直徑捲尺）
   - 攝點間實際距離（用雷射測距儀校正）
5. **拍 baseline 影像**：以 baseline 影像作為「攝點對齊參考」
6. **登記 `fixed_camera_station.csv`**：tree_id、station_id、azimuth_deg、distance_to_trunk_m、camera_height_m、created_at、created_by

#### (二) 每次出場拍攝（重複性）

1. **抵達樣樹**：找到地面標樁
2. **架設相機**：
   - 三腳架置於標樁正上方（誤差 < 5 cm）
   - 相機鏡頭高度依 station 規格設定（用三腳架刻度或捲尺）
   - 鏡頭朝樹幹方向，確認水平（內建水平儀）
3. **拍攝**：
   - 連拍 3 張，取最佳
   - 用相機自動模式 + 固定 ISO（手機可固定 ISO 200~400）
4. **品質快檢**：
   - 整株是否完整入鏡？
   - 焦距是否覆蓋目標？
   - 光照是否過曝 / 過暗？
5. **記錄 metadata**：站別 station_id、camera_id、capture_datetime、實際攝距、實際相機高度

#### (三) 異常處理

| 情況 | 處理 |
|------|------|
| 標樁遺失 | 用 GPS + azimuth + distance 三指標重建，並補打新樁；標記 station_status = `restored` |
| 樹倒 / 枯死 | 標記 tree.status，停止後續攝點拍攝 |
| 樣樹周遭被遮擋（如他樹倒下） | 攝點偏移時必須記錄，分析時加旗標 `obstruction_flag = true` |

### 六、與 ADR-0003 的對接

`fixed_camera_station` 表為 ADR-0003 的試驗設計層實體，欄位定義：

```
fixed_camera_station
├── station_id        TEXT PK         e.g. "115-12-1.T0001.north_side"
├── tree_id           TEXT FK
├── station_label     ENUM            north_side / east_side / leaf_macro / top_view / south_side / west_side / pruning_site_close_up
├── azimuth_deg       DOUBLE          0~360（北=0）
├── distance_to_trunk_m DOUBLE        實測攝距
├── camera_height_m   DOUBLE          相機鏡頭離地高
├── pitch_deg         DOUBLE          相機仰角（水平=0、俯拍=-90）
├── status            ENUM            active / restored / abandoned
├── created_at        DATE
├── created_by        TEXT
└── note              TEXT
```

### 七、命名規則

| 物件 | 規則 | 例 |
|------|------|---|
| station_id | `<site_id>.<tree_id>.<station_label>` | `115-12-1.T0001.north_side` |
| 影像檔名 | `<station_id>_<YYYYMMDD>_<HHMM>_<seq>.<ext>` | `115-12-1.T0001.north_side_20260601_1430_001.jpg` |

> 與 SPEC v0.1 第肆章命名規則衝突，需在 ADR 落地時更新（見 ADR-0003 與本 ADR 共同影響）。

## Consequences

### 一、SPEC 異動

- 第肆章 命名規則：影像檔名規則重寫
- 第拾參章 試驗設計支援（NEW，見 [ADR-0003](0003-schema-alignment-with-xlsx.md)）：固定攝點 SOP 移入正式章節

### 二、SOP 文件異動

- [`docs/pilot-protocol.md`](../pilot-protocol.md) 升 v0.2，含上述章節變更
- [`metadata/field-record-form.md`](../../metadata/field-record-form.md) 加站別欄位

### 三、設備預算影響

| 項目 | 數量估計（300 株） | 單價 | 總額 |
|------|--------------|------|------|
| 金屬木樁（30 cm） | 300 株 × 3 標樁 = **900 支** | NT$15 | NT$13,500 |
| 油漆筆（紅黃藍各 1 組） | 各 6 支 | NT$80 | NT$480 |
| 雷射測距儀（30m） | 1 支 | NT$2,500 | NT$2,500 |
| 三腳架（穩定型） | 2 支 | NT$3,500 | NT$7,000 |
| 羅盤 / 指南針 | 2 個 | NT$500 | NT$1,000 |
| 捲尺（10m） | 2 個 | NT$300 | NT$600 |
| **小計** | — | — | **≈ NT$25,000** |

### 四、人時影響

| 階段 | 估計 |
|------|------|
| 攝點建置（首次） | 每株 30 min × 300 株 = 150 工時 |
| 重複造訪拍攝 | 每株 5 min × 300 株 × 12 場次 = 300 工時/年 |
| **合計第 1 年** | **≈ 450 工時** |

> 建議：攝點建置可分階段，先以 115-12-1（120 株）為 Phase 1A，幼齡林為 Phase 1B。

### 五、訓練資料品質提升

固定攝點帶來的副作用：
- **單張影像可學的特徵更穩定**（攝距、光線、角度都收斂）
- **時序變化單純化**（同攝點的影像差異 ≈ 樹的真實變化）
- **訓練資料規模實際擴大**（每攝點視為一個獨立追蹤目標，不只 300 棵樹）
