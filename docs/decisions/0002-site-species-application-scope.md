> **決議**：TreeVision v0.2 起鎖定為「**114-115 年林業及自然保育署臺中分署土肉桂施肥與修枝矮化試驗監測系統**」，
> 場域為 4 個官方林班作業列區，樹種為土肉桂（*Cinnamomum osmophloeum*），應用主軸為「林下經濟 + 試驗對照監測」。

---

## Context

### 一、Phase 0 v0.1 的範疇模糊問題

[SPEC v0.1](../SPEC.md) 第貳章「目標樹種與場域」對下列三個維度均未定錨：

1. **目標樹種**：寫作「常見人工林樹種」，未指定具體物種
2. **目標場域**：寫作「中海拔人工林」，未指定具體林班 / 管理單位
3. **應用主軸**：寫作「林木健康監測」，未指定產業 / 政策 / 學術哪一條 use case 為優先

此模糊使下游所有設計（影像分類體系、模型架構、健康指標、報表內容）都無法收斂。

### 二、Phase 1 啟動的場域決定

2026-05-04 使用者拍板場域為**林業及自然保育署臺中分署 / 大雪山林業生產合作社**所管轄之林場，並上傳官方計畫文件 4 份（見 [references/source-files-index.md](../../references/source-files-index.md)）。

進一步解讀後發現：此場域**並非自設計試拍場**，而是「**114-115 年林業及自然保育署臺中分署轄區林業永續多元輔導計畫**」之子計畫「**土肉桂施肥與修枝矮化之試驗設計與成效評估**」**已啟動的正式試驗**。試驗設計、樣區、處理組、調查項目、統計方法皆已由官方文件定義（[references/cinnamon-trial-context.md](../../references/cinnamon-trial-context.md) 詳列）。

### 三、決定性意涵

TreeVision 的角色從「為通用林業使用者建構 AI 影像平台」**收斂為**：

> **以影像 AI 強化已既定的官方試驗監測能力**，補上既有調查表（XLSX）難以記錄的影像可量化指標（葉片計數、葉色、病斑、樹冠投影面積等），並對齊試驗對照分析的需求（處理組比較、時序趨勢、ANOVA）。

---

## Options

### 選項 A：保留通用範圍，土肉桂試驗作為 v0 試拍對象

| 屬性 | 內容 |
|------|------|
| SPEC 改動 | 小（僅補一節「v0 試拍場域」） |
| 模型訓練資料 | 仍須涵蓋多樹種（與實際試拍不符） |
| 對齊既有 XLSX schema | 否 |
| 與官方試驗對接成熟度 | 低 |
| 上線可交付性 | 低（範圍太廣） |

### 選項 B：完全收斂為土肉桂試驗監測系統（單一場域、單一樹種、單一應用）

| 屬性 | 內容 |
|------|------|
| SPEC 改動 | 大（第貳～拾貳章都需調整） |
| 模型訓練資料 | 集中、量大（≈ 12,720 張 / 年） |
| 對齊既有 XLSX schema | 是 |
| 與官方試驗對接成熟度 | 高 |
| 上線可交付性 | 高（範圍明確） |
| 風險 | 過度特化，未來擴展其他樹種 / 場域成本上升 |

### 選項 C：分層架構 — 核心鎖定土肉桂試驗，平台層保留多樹種 / 多場域擴展鉤子

| 屬性 | 內容 |
|------|------|
| SPEC 改動 | 中（第貳章鎖定，但第伍章模型架構與第柒章 schema 用泛化設計） |
| 模型訓練資料 | 集中於土肉桂，但 schema 留 species\_id 欄位 |
| 對齊既有 XLSX schema | 是（在 species 欄位寫死土肉桂的前提下） |
| 與官方試驗對接成熟度 | 高 |
| 上線可交付性 | 高 |
| 未來擴展成本 | 低 |

---

## Decision

### 一、選擇 **選項 C**（核心鎖定土肉桂試驗 + 平台層保留擴展鉤子）

理由：

1. **場域與試驗設計已是官方既定事實**，不收斂等於浪費已有資源。
2. **既有 XLSX schema 完整成熟**（10 工作表），TreeVision 對齊即可獲得跨年度可比較的資料基礎。
3. **未來擴展是必然需求**（其他試驗、其他樹種），平台層（schema、API、模型 I/O 介面）保留 species\_id / site\_id 等識別欄位即可，邊際成本低。
4. **避免過度特化的關鍵設計**：
   - schema 中 `tree.species` 欄位保留，但本期專案實際值固定為 `Cinnamomum osmophloeum`
   - 模型權重檔以 `LeafDefect_cinnamon_v1.pt` 等命名標明適用樹種，留下未來訓練 `LeafDefect_general_v1.pt` 的空間
   - 報表模板分為「通用樹木健康履歷」與「試驗對照分析」兩類

### 二、定錨範圍（v0.2 起生效）

| 維度 | 鎖定值 |
|------|-------|
| 場域 | 林業及自然保育署臺中分署轄區 |
| 經營單位 | 大雪山林業生產合作社 |
| 樣區 | 4 個官方林班作業列區（115-12-1 / 117-28-1 / 1-2-3 / 8-1-10） |
| 目標樹種 | 土肉桂（*Cinnamomum osmophloeum* Kanehira） |
| 林齡涵蓋 | 1 個 15 年生成熟林 + 3 個 1~2 年幼齡林 |
| 樣樹規模 | 預估 300 株（120 + 60 × 3） |
| 應用主軸 | 林下經濟（NTFP）試驗對照監測 |
| 試驗設計 | CRBD 雙因子析因（修剪 × 施肥） |
| 處理組合 | 成熟林 2×2 + 幼齡林 2×3 |
| 計畫年度 | 民國 114 年 8 月 1 日 ~ 115 年 8 月 31 日（兩年期，可延伸） |

### 三、平台層擴展鉤子（必須保留）

| 介面 | 擴展鉤子 |
|------|---------|
| `tree_registry.species` | 不寫死，採物種拉丁學名字串 |
| `site_registry.site_type` | 加 enum，未來支援自然林、人工試驗林、行道樹等 |
| 模型權重命名 | `<ModelName>_<species_or_general>_v<n>.<ext>` |
| 報表模板 | 分「通用樹木健康履歷」`tree_default` / 「試驗對照分析」`experiment_compare` |

## Consequences

### 一、SPEC 必改章節（v0.1 → v0.2）

| 章節 | 改動 |
|------|------|
| 第貳章 目標樹種與場域 | 大改 — 寫實鎖定土肉桂 + 4 樣區 |
| 第參章 影像分類體系 | 中改 — 新增「整株（whole_plant）」第三級 |
| 第肆章 命名規則 | 中改 — site_id 對齊官方林班編號；capture_type 新增 `whole`、`scale_ref` |
| 第伍章 模型架構 | 中改 — 新增 PlantStructure（株型解析） |
| 第陸章 健康指標 | 中改 — 拆「健康度 + 試驗響應指標」二線 |
| 第柒章 資料 schema | 大改 — 對齊既有 XLSX 10 工作表（見 ADR-0003） |
| 第捌章 API | 中改 — 新增「試驗對照分析 API」 |
| 第玖章 報表 | 小改 — 新增 `experiment_compare` 模板類型 |
| 第拾貳章 開放問題 | 維持，但新增 7 項待釐清條目（見本 ADR §未解決問題） |

### 三、營運面影響

| 項目 | 影響 |
|------|------|
| Firebase 專案 | `treevision-data` 必須**啟用 Blaze plan**（年估 12,720 張 / 5 GB+） |
| 資料治理 | 影像中可能含合作社經營細節，**公開層需脫敏處理**（依 ADR-0001 議題 5 GPS 三層遮蔽政策） |
| 計畫合規 | 所有後續輸出需符合「林業永續多元輔導計畫」交付規格 |
| 智慧財產 | 計畫期間之衍生模型權重之歸屬須與委託單位確認（見未解決問題 7） |

| 項目 | 原定 | 異動為 |
|------|------|------|

| Phase 0.5 | （無） | **新增**：v0.2 規格大改（2026-05） |
| Phase 1 啟動 | 2026-05 | 2026-06（v0.2 規格收尾後） |
| Phase 1 影像目標 | 樹冠 ≥ 500 張、葉片 ≥ 1,000 張 | **≈ 12,720 張 / 年**（依官方試驗 300 樣樹估算） |
